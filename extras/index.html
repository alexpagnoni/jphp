What's new in JPHP 2.1 ?
  1. Class and package loader has been improved to get better performance and offering a nice view of the entire package tree
  2. Some new packages and classes appears :
    1. jphp.xml : A package dedicated to XML operation, including a DOM parser
    2. jphp.security : contain class using for authentication system etc.
    3. jphp.fastsql : provide a easiest way to query sql to a database
    4. jphp.sql : if you are a v0.5b user, this package's still teh same ... new drivers and improvement will be available soon.
    5. jphp.io : this package is dedicated to IO operation that include many classes and interfaces to base on. Exemple, now the package provide : FileReader/FileWriter, ObjectReader/ObjectWriter ...
    6. jphp.datastructure : provide base complexe data structure to subclass depending on your need
    7. jphp.util : some new class has been added and other has been improved : ObjectSerializer (simple class to easy serialization), Comparator (interface), Arrays (sorts etc.)
    8. jphp.http : contains many interfaces related to http (request, session, context, cookie.


About jphp.http package :
jphp.http package you will find those classes :
  1. Cookie : a class that represent a cookie
  2. UploadFile : an extension to File object to represent an file that has been uploaded via HTTP POST
  3. HttpContext : a simple Context manager using file, and storing object that must be share by all user
  4. HttpRequest : a encapsulate http request to make more user friendly and object oriented access to http request
  5. HttpResponse : this class simple represent a http response providing method to add cookie using cookie object and header method
  6. HttpSession : a simple proprietary session manager

jphp.http.interface provide base interface that are use in all classes above, and you can reuse this to provide your own class :
  1. IHttpRequest : basic interface to manipulate Request
  2. IHttpResponse : basic interface to manipulate Response
  3. IHttpContext : basic interface to manipulate Context
  4. IHttpSession : basic interface to manipulate Session

jphp.http.servlet provide an web application configuration using XML file, if you are familiar with java web.xml config file. So this package should be very interesting as I'll use it to implement the MVC framework. All classes you will find here extends from classes found in jphp.http package to add communication between Context, Session, Request and Response.
Example, a request object should be able to create/load a session from user cookie information.
Check the demo to see how it is used

jphp.io, provide and easy and elegant input/output API using Reader/Writer and FilterReader and FilterWriter interface. Extends classes or interface to extends functionnality. Exemple, I've extends FilterWriter to create a ObjectWriter providing a simpleway to write many object stream into a targeted Writer object. And a ObjectReader extends the FilterReader to stream the object writed by the ObjectWriter.

1) Extract jphp.2.1.alpha-r1.zip anywhere under your document root directory.
